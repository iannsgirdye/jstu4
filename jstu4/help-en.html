Help

Command format

q,a,v,q', where:
  • q — current state;
  • a — symbol currently seen on a tape;
  • v — command or symbol to write;
  • q' — new state.

Available commands

  • > — move right;
  • < — move left;
  • = — don't move at this turn;
  • # — stop the machine.

Multiple commands in one line

You can write several commands in one separating them with any number of spaces.

Comments

C-style (//) or shell-style comments can be added to program. Note: there must be at least one space/linebreak between a command and a comment.

Named states

Named states are supported. State names which consist of digits only are interpreted as decimal numbers, so 01 and 1 represent the same state.

Execution process

While program is running commands are marked with different color. You can clear those marks by clicking the "Unmark" button

Tape

The tape is limited at the left edge.

Buttons description

  • — parse program and run it;
  • — don't wait for user to click "Step", just run. When clicked again: run faster;
  • — perform the next step (quick mode will be canceled if active);
  • — stop program and return to edit mode;
  • — convert program to the TU4 format (replace state names with hex numbers: 0x00 - 0xFF).

Hotkeys

  • Ctrl + Enter in edit mode — run program (also Enter in the "Tape" field will do);
  • Enter or Ctrl + Enter in run mode — toggle quick run;
  • Space in run mode — perform the next step;
  • Esc in run mode — return to the edit mode;

Example

Example (will replace 0 to 1 and 1 to 0 in place):
00, ,<,rep rep,0,1,mov rep,1,0,mov rep, ,>,fin // rewrite cell mov,0,<,rep mov,1,<,rep # do a step to the left fin,0,>,fin fin,1,>,fin fin, ,#,fin // finally, move to the right
